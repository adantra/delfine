\section{FEniCS/Dolfin}
\label{sc:dolfin}
Para a geração do código responsável pela montagem das matrizes e vetores necessários para a análise via \ac{MEF} do problema foi utilizada a biblioteca \emph{FEniCS/DOLFIN} \citep{Logg2010}, a qual permite ao desenvolvedor escrever as formas lineares e bilineares apresentadas na Eq. (\ref{eq:galerkin}) usando um script escrito em código \emph{Python} de alto nível, o qual se assemelha à notação matemática, sendo este script convertido automaticamente em código \emph{C++} otimizado, o qual pode ser executado de modo bastante efetivo para problemas gerais.

\section{PyAMG}
\label{sc:pyamg}
O sistema de equações obtido da discretização com o auxílio da biblioteca \emph{FEniCS/DOLFIN} é resolvido neste trabalho utilizando uma abordagem via método Multigrid Algébrico \ac{AMG} implementado no pacote \emph{PyAMG} \citep{Bell2008}, o qual disponibiliza o acesso através de uma interface em \emph{Python} a vários tipos de algoritmos multigrid já implementados. As rotinas deste pacote que demandam a maior capacidade de processamento são implementadas e compiladas em \emph{C++} por motivo de performance, porém o usuário não precisa lidar com esta interface em código de baixo nível.
